## Executive summary

Across 36 head-to-head closed-loop cells (throughput): **Rust faster in 10**, **Java faster in 19**, **within noise in 7**.


Per suite (throughput wins R / J / noise): **A** 4/11/2; **B** 4/1/5; **J** 2/7/0.


**Resource efficiency (the largest, most consistent gap):** Rust's peak RSS is a median **0.1×** of Java's (i.e. Java uses ~18× more memory), and Rust's CPU-per-op is a median **0.59×** of Java's (~1.7× less CPU per op). Both clients ran with **zero errors**.


Most decisive throughput cells: Java — imap.set.default C256 v100 uniform (×1.32 J), imap.put.update C256 v100 uniform (×1.28 J); Rust — cpmap.put.update C1 v8 uniform (×1.80 R), atomiclong.compare_and_set.success C1 v8 uniform (×1.36 R).


> Honesty contract: every figure is a **median across forks×trials with a 95% bootstrap CI**. Cells whose Rust/Java ratio CI spans 1.0 are flagged **within noise** and no winner is claimed. Tail percentiles report their sample count. The rig is **co-located** (see Caveats). CPU-per-Mops is a whole-process approximation (mean sampled CPU percent × measure window ÷ ops).

## Caveats (fairness deviations, stated plainly)

- **Co-located rig.** Cluster + client share one host with disjoint core pinning (members `0-3,8-11`, client `4-6,12-14`). They still share L3/memory-bandwidth/loopback; absolute throughput is higher than a real LAN deployment but BOTH clients see identical conditions, so the comparison is fair.

- **CPU governor** is not settable in the virtualized guest (no cpufreq); frequency is hypervisor-managed. Achieved MHz is recorded in provenance.

- **Java client = 5.7.0** (latest 5.7.x on Maven Central), version-identical to the EE 5.7.0 cluster — exact protocol parity. JVM: Corretto 21. JFR uses low-overhead `default` settings; GC logging adds negligible overhead.

- **Wire parity:** byte[]/long/String values only (no Compact/Portable), identical Data framing, so per-op wire bytes match; smart routing ON, statistics OFF, near-cache OFF for both.

- **net bytes/op** is measured on loopback (counts both directions); treat as a relative framing indicator, not an absolute on-wire byte count.

- Raw per-run records + embedded HdrHistograms are preserved in the run directory (auditable).
